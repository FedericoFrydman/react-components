# Form Field

The FormField component is a container that adds the labels; validation messages, hints and colors for the different controls in a form (inputs, selects, checkbox, radio).

<Alert title="Dev Note" severity="warning" icon="alert_warning">
  Based on the text above, Input component should always be used as child of FormField.
</Alert>

**About the input:**

- Input allows people to enter data for further use in your product, or to configure options. Inputs and forms are used to get information and guide people with minimal fuss.
- Input should always be accompanied by a label for accessibility purposes with the exception of Search (for this reason, the default text color for Search is N1000 for color contrast.

## Usage guidelines

- **Do** include a label with all inputs. Label should be sentence case and keep it short
- **Do not** use placeholder as label text
- **Do** use placeholder text to give hints what should go into the input/field
- **Do** make it clear with an \* asterisk if an individual input is required
- **Do** mark "Optional" for optional inputs when majority of other inputs are required
- **Do** include description if an input format is expected or give context why we are collecting the information from users

## Example Code

```tsx
export const EmailForm: FC = ({}) => {

  ...

  return (
    <Form onSubmit={submitForm}>
      <FormField
        id="email"
        status={email.status}
        validationHint={email.hint}
        label="Email Address"
        type="input"
        required
      >
        <Input
          name="email"
          onChange={handleInputChange}
          placeholder="Email Address"
          value={email.value}
          pattern={EMAIL_PATTERN}
          type="email"
          required
        />
      </FormField>
      <Button
        type="submit"
        disabled={!isValid}
        label="Send Email"
        classNames={['mt-5']}
        stretch
      />
    </Form>
  )
}
```
